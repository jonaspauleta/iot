# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

M5Stack Core (Basic) desk gadget that displays live coding-plan usage for Claude, Codex, Cursor, and Grok. The **Mac does all the work** (`mac/`): reads each provider's local credentials, fetches usage from their internal endpoints, normalizes to a compact JSON frame, pushes it over USB serial. The **device is a pure display** (`firmware/`): pages between provider windows, renders bars, extrapolates reset countdowns.

## Runtimes

- `mac/` runs on **Bun 1.3+**, not Node. It has **no `node_modules` and no dependencies** by design (built-in `fetch`, `Bun.spawn`, `node:fs`). Do not add dependencies or reach for `npm install`. The `serialport` package crashes under Bun, which is why serial is hand-rolled with `stty` + `fs`.
- `firmware/` is C++/Arduino on ESP32 via **PlatformIO** (`M5Unified`, `ArduinoJson`).

## Commands

```bash
cd mac && bun start          # run the pusher (reads creds, polls, pushes over serial)
cd mac && bun push.js --stdout   # print one real frame to terminal (no serial)
cd mac && bun push.js --replay   # cycle synthetic fixture frames (no network)
cd mac && bun run test       # run all pure-function tests
cd firmware && pio run -t upload # flash the firmware
```

## Running in the background (LaunchAgent)

`./pusher.sh` (repo root) manages the pusher as a macOS LaunchAgent
(`com.jonaspauleta.m5usage`), so it auto-starts at login and restarts on crash.

```bash
./pusher.sh start    # install/refresh the plist + load it
./pusher.sh stop     # unload it (frees the serial port, e.g. before reflashing)
./pusher.sh restart
./pusher.sh status   # agent state + who holds the serial port
./pusher.sh logs     # tail ~/Library/Logs/m5-usage.log
```

- The plist lives at `~/Library/LaunchAgents/com.jonaspauleta.m5usage.plist`,
  regenerated by `start` (not in the repo). It runs `bun push.js` from `mac/`.
- It is a LaunchAgent, not a Daemon, so it runs in the GUI session and can read
  the login Keychain. First run may prompt for "confidential information"
  (Claude/Cursor tokens), click Always Allow or those windows show `reauth`.
- Only one process may hold the serial port: do not also run `bun start` manually
  while the agent is loaded. Run `./pusher.sh stop` before flashing, then `start`.

## Serial gotchas (the fragile part)

- **Two baud rates:** flash/upload is **460800**, runtime is **115200**. Do NOT raise upload to 921600, it causes serial noise on this board's CP2104 chip.
- **Hold the port open.** Opening the serial port resets the ESP32 (DTR). `push.js` opens once and holds it; reopening per poll would reset the device each poll. A `resend()` re-feeds the cached frame every ~3s so the device self-heals after a reset/reboot.
- **stty runs after `fs.openSync`, not before** (`lib/serial.js`). A fresh fd inherits whatever rate the driver was last set to (e.g. 460800 from an esptool upload), so setting baud before open does not stick.
- **Port contention:** do not run a PlatformIO serial monitor while `bun start` is running. `Resource temporarily unavailable` on flash means the pusher holds the port, stop it, flash, restart.
- Keep `POLL_MS` multi-minute (default 300000 = 5 min); provider endpoints throttle frequent polling.

## Conventions

- Each provider (`mac/providers/*.js`) splits a **pure `parseX(json) -> bars`** function (no I/O, marked `// PURE:`) from the fetch/credentials layer. Keep this split when editing or adding providers, and add a co-located `*.test.js`.
- Provider endpoints are **reverse-engineered and undocumented**; expect breakage if a response shape changes.
- Keychain reads must use the literal `/usr/bin/security` binary (some items, e.g. Cursor, pin their ACL to it; any other reader triggers a GUI prompt). Only Codex's `auth.json` is ever written back (atomic, 0600, for token refresh); all other tokens are read-only.
- JS: 2-space indent, semicolons, single quotes, CommonJS. No linter/formatter is configured; match surrounding style.

## Env vars

`PORT` (pin serial port, else auto-detects `/dev/cu.usbserial*`), `POLL_MS`, `CODEX_HOME`, `GROK_HOME`.
